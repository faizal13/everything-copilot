# Golang Patterns Skill

## Name
Golang Patterns

## Description
Go-specific patterns for writing idiomatic, maintainable Go code. Covers interfaces, testing, concurrency, and error handling following Effective Go principles.

## Trigger Conditions
- Working on Go source files (`.go`)
- Go module management (`go.mod`, `go.sum`)
- Go build or test commands
- Go-specific code review

## Files
- `idiomatic-go.md` — Effective Go patterns, naming, package layout
- `go-interfaces.md` — Interface design, implicit implementation, composition
- `go-testing.md` — Table-driven tests, benchmarks, fuzzing
- `go-concurrency.md` — Goroutines, channels, sync, context, errgroup

## Model Recommendation
- **Sonnet** for Go implementation and code review
- **Opus** for Go architecture and concurrency design
